WDD 330: Week 1 Notes

How do I store data so it isn't lost when the application stops?

Data can easily be stored in the users local browser using the "localStorage.SetItem()" function. I have used this many times in past applications and it can be very useful. There wasn't really anything new here for me to learn although you can see the example I created in the "Story Editor" in the table of contents.

How do I get data back into my application once it has been stored?

Data can be easily retrieved from the localStorage using the "localStorage.GetItem()" functioln. Once again I have used this before and didn't have much more than a brief review of the topic.

What can I do to allow the user to communicate?

I'm not sure what this question is referring to, but I'm guessing it means through inputs. I can give them text boxes and inputs that accept information in all kinds of ways but also help limit the type of response the user is able to give. This makes the input more easily usable and testable.
