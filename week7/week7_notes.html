Chapter 11: Further Functions

Something New: Call- Functions can use the 'this' key word. If I then say 'myFunction.call(objectNamedBruce);' then the this key word will refer to that object. It is much like using the function as a method of the object. If the function has parameters these must be placed after the object. For instance, 'myFunction.call(objectNamedBruce, param1, param2);' You can still use call even if there isn't a 'this' key word in it, you just have to pass in a null for the object. For instance, 'myFunction.call(null, param1, param2);'

Apply(): Similar to Call, this funtion allows you to call functions but you pass in the parameters as an array. For instance, 'myFunction.apply(null, [param1, param2];'

Custom Properties: Custom functions can have properties, just like an object. You can say, 'myFunction.property = "I have a new property";'

Memoization: Using the custom properties mentioned above we can cache answers if we have a time consuming function. just use 'functionName.cache[input] = something'. The Next time it will return the result faster.

IIFE functions: defined and immidiately called. (function(){something}'.

Recursive Functions: functions that return themselves and put in a new input until the function reaches a certain point.

I've used both so I won't get into it, but callbacks and promises are both available in js as are Async functions.

Generator Functions: instead of running the code inside the function, a generator object is returned. This can be used to create things like sequences.

Pure functions: return a value that only depends on the input values and does not change the value of the input variables.

High Order Functions: functions that accept a function as a parameter or return a function.

Chapter 13: AJAX

Intro: Ajax is a technique that allows web pages to communicate asynchronously with a server and it dynaically updates web pages without reloading.

The fetch API: Used to gt asyncronous information accross websites.

Redirects: Redirect() methos can be used to redirect to another URL. There is no support for redirect() in any browser.

Blob: The blob() method is used to read a file of raw data such as an image or a spreadsheet. It returns a promise that resolves with a blob object.

Request: We can provide a request object to get more fine-grained control over the request being made. Request objects, once made, can be passed in as the parameter to the fetch call.

HEaders: the fetch api introduced a "headers" interface which can be used to create a headers object. Headers usuallyu resolve things like content type and coors information.

Sending Information: Ajax can also be used to send information. Usually we do this as a json string.

Form Data: The fetch API includes a formdata interface which makes it easier to submit information in forms using Ajax. It can be used to submit files as well.
